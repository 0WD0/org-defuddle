* Cloud Sessions

:DESCRIPTION: How cloud sessions work and what is available inside them.
:LANGUAGE: en
:WORD_COUNT: 96
:END:

Cloud sessions start from a fresh clone of your repository. Anything committed to the repo is available. Anything you've installed or configured only on your own machine is not.

Sessions run on managed cloud infrastructure. You can monitor them from the mobile app, and they persist even if you close your browser. See the [[https://example.com/docs/get-started][getting started guide]] for more.

** What is available

Each session includes the tools needed to build, test, and commit changes. The environment is isolated per session.

You can extend the environment with a setup script. The script runs once at session start.
